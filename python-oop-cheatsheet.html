What is Object-Oriented Programming?

OOP is a way of writing programs by modeling real-world things. Instead of just writing step-by-step instructions, you create "objects" that have properties (data) and can do things (actions).

OOP = Model the real world in code

Class & Object

A Class is a blueprint (template). An Object is a real thing built from that blueprint.

One blueprint → many objects, each with its own data
Python Code
class Car:            # This is the BLUEPRINT
    pass

my_car   = Car()      # Object 1 🚗
your_car = Car()      # Object 2 🚙
boss_car = Car()      # Object 3 🏎️
🌍 Real-World Analogies

  • Object = Cookie — each one is made from the same cutter but can have different toppings

Attributes & Methods

Attributes = what the object has (properties/data)
Methods = what the object can do (actions/functions)

class Dog:
    def __init__(self, name, breed):
        self.name  = name    # Attribute
        self.breed = breed   # Attribute

    def bark(self):         # Method
        print(f"{self.name} says Woof!")

buddy = Dog("Buddy", "Golden Retriever")
print(buddy.name)    # Buddy
buddy.bark()         # Buddy says Woof!
🌍 Real-World Analogy

A phone HAS a brand, color, battery (attributes). It can call(), text(), take_photo() (methods).

The Constructor: __init__

The __init__ method runs automatically when you create a new object. It sets up the initial data.

__init__ is like a factory — feeds data into a brand-new object
class Student:
    def __init__(self, name, age, grade):
        self.name  = name   # Store name
        self.age   = age    # Store age
        self.grade = grade  # Store grade

# Creating objects — __init__ runs automatically!
alice = Student("Alice", 14, "9th")
bob   = Student("Bob",   15, "10th")

print(alice.name)   # Alice
print(bob.grade)    # 10th
🌍 Real-World Analogy

Like filling out a birth certificate — the moment a baby is born, you record their name, date, weight. The __init__ method does the same when an object is "born."

📋 More Examples of __init__ in Action
  • Opening a new bank account → you fill out a form with name, address, initial deposit. That's __init__!
  • Signing up for Netflix → you enter email, password, plan type. All stored when your account object is created.
  • Ordering Uber → pickup location, destination, ride type are set the moment you book = __init__

The "self" Keyword

self refers to the specific object that is calling the method. It's how each object keeps track of its own data.

class Cat:
    def __init__(self, name):
        self.name = name   # "MY name is..."

    def introduce(self):
        print(f"I am {self.name}")

whiskers = Cat("Whiskers")
mittens  = Cat("Mittens")
whiskers.introduce()  # I am Whiskers
mittens.introduce()   # I am Mittens
🌍 Real-World Analogy

self is like your name tag at a party. When you say "My name is..." you're talking about yourself, not someone else.

Encapsulation

Encapsulation = Bundling data and methods together, and hiding internal details so no one can mess things up.

Data is locked inside — only methods provide safe access
class BankAccount:
    def __init__(self, owner, balance):
        self.owner     = owner
        self.__balance = balance   # 🔒 Private!

    def deposit(self, amount):
        self.__balance += amount
        print(f"Deposited ${amount}")

    def get_balance(self):
        return self.__balance

acct = BankAccount("Alice", 5000)
acct.deposit(500)
print(acct.get_balance())  # 5500
# acct.__balance  ← ❌ Error! Can't access directly
💡 Python Convention: Use __ (double underscore) prefix to make attributes private. Use _ (single underscore) for "please don't touch" (protected).
🌍 Real-World Analogies
  • ATM machine — you can deposit and withdraw money (methods), but you can't directly reach into the vault to grab cash (private data)
  • Medicine capsule — the medicine is sealed inside. You take the whole capsule; you don't open it and pick individual chemicals
  • TV remote — you press buttons (public methods). The circuit board inside is hidden (private data). You don't need to solder wires to change the channel!
  • Microwave — you set time and press Start. You don't control the magnetron, voltage, or radiation levels directly

Inheritance — "Child Gets Traits from Parent"

A new class (child) can inherit all the attributes and methods from an existing class (parent), and add its own unique features.

Children inherit everything from the parent, plus add their own special abilities
class Animal:                          # Parent class
    def __init__(self, name):
        self.name = name
    def eat(self):
        print(f"{self.name} is eating")

class Dog(Animal):                      # Child inherits from Animal
    def bark(self):
        print(f"{self.name} says Woof!")

class Cat(Animal):                      # Another child
    def purr(self):
        print(f"{self.name} purrs...")

rex = Dog("Rex")
rex.eat()     # Rex is eating   ← inherited!
rex.bark()    # Rex says Woof!  ← its own method
🌍 Real-World Analogies
  • Family tree — you inherit eye color, height from parents. But you also develop your own unique talents
  • Vehicles — All vehicles have wheels & engine. A Truck adds cargo capacity. A Bus adds passenger seats. A Motorcycle changes wheel count
  • Smartphones — All phones can call & text (parent). iPhone adds FaceTime. Android adds widgets. Samsung adds S-Pen
  • Social media — All posts have author & timestamp. A PhotoPost adds image. A VideoPost adds duration. A StoryPost adds expiry time
📞

super() — Calling the Parent

super() lets a child class call a method from its parent class. Useful when you want to extend (not replace) what the parent does.

class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)  # Call parent's __init__
        self.breed = breed     # Add own attribute

rex = Dog("Rex", "Labrador")
print(rex.name)    # Rex (from parent)
print(rex.breed)   # Labrador (own)
🌍 Real-World Analogy

Like calling your parents for advice before making your own decision — super() says "Hey Parent, do your thing first, then I'll add mine."

Polymorphism

Polymorphism = "many forms." The same method name behaves differently depending on the object.

Same method name "speak()" — each animal does it differently!
class Dog:
    def speak(self):
        return "Woof!"

class Cat:
    def speak(self):
        return "Meow!"

class Duck:
    def speak(self):
        return "Quack!"

# Same function works with ANY animal!
for animal in [Dog(), Cat(), Duck()]:
    print(animal.speak())
Woof!
Meow!
Quack!
🌍 Real-World Analogies
  • TV remote "play" button — press it on Netflix, YouTube, or Spotify. Same button, different content plays!
  • "Pay" at checkout — works with cash, credit card, Apple Pay, UPI. Same action, different payment process each time
  • "Print" command — same command prints to a laser printer, inkjet, or PDF. Each does it differently
  • "Start" a vehicle — car uses a key, Tesla uses a button, motorcycle uses a kick. Same concept, different implementation
🎨

Abstraction

Abstraction = Hide the complex stuff, show only what's needed. You define "what to do" but each child decides "how to do it."

from abc import ABC, abstractmethod

class Shape(ABC):            # Abstract class
    @abstractmethod
    def area(self):
        pass                  # No code here — children MUST implement

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    def area(self):
        return 3.14 * self.radius ** 2

class Rectangle(Shape):
    def __init__(self, w, h):
        self.w, self.h = w, h
    def area(self):
        return self.w * self.h

c = Circle(5)
print(c.area())        # 78.5
🌍 Real-World Analogy

A car dashboard — you just press Start, Gas, Brake. You don't need to know how the engine, fuel injection, or transmission work inside!

Method Overriding

A child class can replace (override) a method from the parent class with its own version.

class Animal:
    def speak(self):
        print("Some sound...")

class Dog(Animal):
    def speak(self):          # Override!
        print("Woof Woof!")

pet = Dog()
pet.speak()   # Woof Woof!  (not "Some sound...")
🌍 Real-World Analogy

Your family recipe for cookies — your mom has her version, but you override it with your own twist (extra chocolate chips!).

Magic (Dunder) Methods

Methods with double underscores (__name__) are "magic" — Python calls them automatically in special situations.

MethodWhen Python Calls ItExample
__init__Creating a new objectobj = MyClass()
__str__Printing the objectprint(obj)
__len__Getting lengthlen(obj)
__eq__Comparing with ==obj1 == obj2
__add__Using + operatorobj1 + obj2
__repr__Developer-friendly displayrepr(obj)
class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    def __str__(self):
        return f"📖 {self.title} ({self.pages} pages)"

    def __len__(self):
        return self.pages

book = Book("Python 101", 350)
print(book)       # 📖 Python 101 (350 pages)
print(len(book))  # 350
🌍 Real-World Analogy

Magic methods are like automatic reflexes. When someone asks "how tall are you?" you don't think — you just answer. Similarly, when Python asks len(book), the __len__ method fires automatically.

Class vs Instance Variables

class Dog:
    species = "Canis familiaris"  # Class variable (shared)

    def __init__(self, name):
        self.name = name          # Instance variable (unique)

rex = Dog("Rex")
luna = Dog("Luna")

print(rex.species)    # Canis familiaris (shared)
print(luna.species)   # Canis familiaris (shared)
print(rex.name)       # Rex   (unique)
print(luna.name)      # Luna  (unique)
🌍 Real-World Analogy

Class variable = School name (same for all students). Instance variable = Student's name (different for each student).

Complete Example: Online Shopping System

Let's build a mini shopping system using all the OOP concepts you just learned!

class Product:                             # 📦 Base class
    def __init__(self, name, price):
        self.name  = name
        self.price = price

    def display(self):
        print(f"{self.name} - ${self.price}")

class Electronics(Product):                 # 📱 Inheritance!
    def __init__(self, name, price, warranty):
        super().__init__(name, price)       # super()!
        self.warranty = warranty

    def display(self):                      # Polymorphism!
        print(f"{self.name} - ${self.price} ({self.warranty} warranty)")

class Cart:                                 # 🛒 Encapsulation!
    def __init__(self):
        self.__items = []                   # Private list

    def add(self, product):
        self.__items.append(product)
        print(f"Added {product.name} to cart")

    def total(self):
        return sum(item.price for item in self.__items)

    def show(self):
        for item in self.__items:
            item.display()                 # Polymorphism in action!

# 🎮 Let's shop!
phone  = Electronics("iPhone", 999, "2yr")
book   = Product("Python Book", 35)

cart = Cart()
cart.add(phone)
cart.add(book)
cart.show()
print(f"Total: ${cart.total()}")
Added iPhone to cart
Added Python Book to cart
iPhone - $999 (2yr warranty)
Python Book - $35
Total: $1034

Real-World Example: Pizza Ordering System

Let's model a pizza shop — showing inheritance, polymorphism, and encapsulation in one tasty example!

class Pizza:
    def __init__(self, size):
        self.size = size    # "small", "medium", "large"

    def price(self):       # Base price by size
        prices = {"small": 8, "medium": 10, "large": 12}
        return prices[self.size]

    def describe(self):
        print(f"A {self.size} pizza - ${self.price()}")

class PepperoniPizza(Pizza):           # 🧬 Inheritance!
    def price(self):                    # 🎭 Polymorphism!
        return super().price() + 5     # 📞 super() adds base + $5

    def describe(self):
        print(f"🥩 {self.size} Pepperoni - ${self.price()}")

class VeggiePizza(Pizza):
    def __init__(self, size, toppings):
        super().__init__(size)
        self.toppings = toppings

    def describe(self):
        print(f"🥦 {self.size} Veggie ({', '.join(self.toppings)}) - ${self.price()}")

# Order up!
order = [PepperoniPizza("large"), VeggiePizza("medium", ["mushrooms", "olives"])]
for pizza in order:
    pizza.describe()
🥩 large Pepperoni - $17
🥦 medium Veggie (mushrooms, olives) - $10

Real-World Example: Library System

A library manages books, members, and borrowing — a perfect fit for OOP with encapsulation and methods.

class Book:
    def __init__(self, title, author):
        self.title     = title
        self.author    = author
        self.available = True

    def __str__(self):
        status = "✅ Available" if self.available else "❌ Borrowed"
        return f"📖 {self.title} by {self.author} [{status}]"

class Member:
    def __init__(self, name):
        self.name = name
        self.__borrowed = []       # 🔒 Private!

    def borrow(self, book):
        if book.available:
            book.available = False
            self.__borrowed.append(book)
            print(f"{self.name} borrowed '{book.title}'")
        else:
            print(f"Sorry, '{book.title}' is not available!")

    def return_book(self, book):
        book.available = True
        self.__borrowed.remove(book)
        print(f"{self.name} returned '{book.title}'")

# 📚 Let's use it!
harry_potter = Book("Harry Potter", "J.K. Rowling")
alice = Member("Alice")

print(harry_potter)         # 📖 Harry Potter by J.K. Rowling [✅ Available]
alice.borrow(harry_potter)  # Alice borrowed 'Harry Potter'
print(harry_potter)         # 📖 Harry Potter by J.K. Rowling [❌ Borrowed]
🌍 See the OOP concepts?
  • Encapsulation — __borrowed list is private; members can only borrow/return through methods
  • __str__ magic method — books display themselves beautifully when printed
  • Objects interact — a Member object uses a Book object. That's how real systems work!

Real-World Example: Social Media

Model Instagram-like posts with inheritance!

class Post:                            # Base post
    def __init__(self, author, caption):
        self.author  = author
        self.caption = caption
        self.likes   = 0

    def like(self):
        self.likes += 1

    def display(self):
        print(f"@{self.author}: {self.caption} (❤️ {self.likes})")

class PhotoPost(Post):                # 📷 Adds image
    def __init__(self, author, caption, photo):
        super().__init__(author, caption)
        self.photo = photo

    def display(self):
        print(f"📷 @{self.author}: {self.caption} [Photo: {self.photo}] (❤️ {self.likes})")

class ReelPost(Post):                 # 🎬 Adds video
    def __init__(self, author, caption, duration):
        super().__init__(author, caption)
        self.duration = duration

    def display(self):
        print(f"🎬 @{self.author}: {self.caption} [{self.duration}s] (❤️ {self.likes})")

Real-World Example: Game Characters

Video games are built with OOP! Every character, weapon, and level is an object.

class Character:
    def __init__(self, name, health=100):
        self.name   = name
        self.health = health
        self.alive  = True

    def take_damage(self, amount):
        self.health -= amount
        if self.health <= 0:
            self.alive = False
            print(f"💀 {self.name} has been defeated!")
        else:
            print(f"💔 {self.name} HP: {self.health}")

class Warrior(Character):
    def attack(self, target):
        print(f"⚔️ {self.name} slashes {target.name}!")
        target.take_damage(30)

class Mage(Character):
    def attack(self, target):
        print(f"🔮 {self.name} casts fireball at {target.name}!")
        target.take_damage(50)

hero    = Warrior("Arthur")
villain = Mage("Morgana")

hero.attack(villain)
villain.attack(hero)
⚔️ Arthur slashes Morgana!
💔 Morgana HP: 70
🔮 Morgana casts fireball at Arthur!
💔 Arthur HP: 50

Common Beginner Mistakes

❌ Forgetting self
Every method's first parameter must be self.
def bark(): → def bark(self):
❌ Forgetting self. when accessing attributes
print(name) → print(self.name)
❌ Not calling super().__init__()
If your child class has __init__, call super().__init__() to initialize parent attributes.
❌ Using mutable default class variables
class Dog: tricks = [] — this list is shared by ALL dogs! Use self.tricks = [] in __init__ instead.
